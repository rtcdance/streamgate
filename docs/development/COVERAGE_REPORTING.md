# Code Coverage Reporting in CI/CD

## Overview

The StreamGate CI/CD pipeline now includes comprehensive code coverage reporting with the following features:

## Coverage Collection

### Test Types with Coverage

1. **Unit Tests** (`coverage-unit.out`)
   - Runs: `go test -v -race -coverprofile=coverage-unit.out ./test/unit/...`
   - Coverage: Individual component testing

2. **Integration Tests** (`coverage-integration.out`)
   - Runs: `go test -v -race -coverprofile=coverage-integration.out ./test/integration/...`
   - Coverage: Service integration testing

3. **E2E Tests** (`coverage-e2e.out`)
   - Runs: `go test -v -race -coverprofile=coverage-e2e.out ./test/e2e/...`
   - Coverage: End-to-end workflow testing

4. **Load Tests** (`coverage-load.out`)
   - Runs: `go test -v -race -coverprofile=coverage-load.out -timeout=30m ./test/load/...`
   - Coverage: Load scenario testing

5. **Benchmark Tests** (No coverage profile)
   - Runs: `go test -bench=. -benchmem -run=^$ ./test/benchmark/...`
   - Purpose: Performance benchmarking

## Coverage Merging

### Tool: gocovmerge

All coverage profiles are merged using `gocovmerge`:
```bash
go install github.com/wadey/gocovmerge@latest
gocovmerge coverage-unit.out coverage-integration.out coverage-e2e.out coverage-load.out > coverage-total.out
```

## Coverage Reports

### 1. HTML Report

**File:** `coverage.html`
- Generated by: `go tool cover -html=coverage-total.out -o coverage.html`
- Purpose: Interactive HTML coverage visualization
- Features:
  - Color-coded coverage (green: covered, red: uncovered)
  - Click-to-navigate to source code
  - Percentage indicators per file

### 2. Function Coverage Report

**File:** `coverage-func.txt`
- Generated by: `go tool cover -func=coverage-total.out -o coverage-func.txt`
- Purpose: Detailed function-level coverage
- Format:
  ```
  github.com/streamgate/pkg/eventbus/eventbus.go:25:    Init        100.0%
  github.com/streamgate/pkg/eventbus/eventbus.go:32:    Subscribe    95.5%
  ...
  total:                                    (statements) 85.3%
  ```

### 3. Coverage Badge

**File:** `coverage-badge.svg`
- Generated by: Custom SVG generation
- Purpose: Visual coverage indicator for README/badges
- Features:
  - Color-coded based on coverage percentage
  - SVG format for easy embedding
  - Displays exact percentage

## Coverage Threshold

### Minimum Coverage: 80%

The CI pipeline enforces a minimum coverage threshold of 80%:
- **Below 80%**: CI fails with error message
- **80% and above**: CI passes successfully
- **Warning**: Coverage below threshold generates warning in GitHub summary

## Coverage Reporting

### GitHub Summary

Coverage is reported in GitHub Actions summary with:

1. **Overall Coverage Percentage**
   - Total coverage across all test types
   - Threshold status (✅ meets / ⚠️ below)

2. **Coverage by Package**
   - Table showing coverage per package
   - Format: `| Package | Coverage |`

3. **Coverage Details**
   - Full function-level coverage breakdown
   - Sorted by package

### PR Comments

For pull requests, coverage is automatically commented with:

```markdown
## 📊 Code Coverage Report

### Overall Coverage: 85.3%

<details>
<summary>View detailed coverage</summary>

```
See coverage report artifact for detailed breakdown
```
</details>

---

📈 Coverage Badge: ![coverage](https://img.shields.io/badge/coverage-85.3%25-brightgreen)

📄 Full Report: [View HTML](https://github.com/owner/repo/actions/runs/123456)
```

## Coverage Artifacts

All coverage reports are uploaded as GitHub Actions artifacts:

1. **coverage-report** artifact containing:
   - `coverage-total.out` - Merged coverage profile
   - `coverage.html` - HTML visualization
   - `coverage-func.txt` - Function coverage details
   - `coverage-badge.svg` - Coverage badge

## Coverage Deployment

### GitHub Pages

Coverage HTML report is deployed to GitHub Pages:
- **URL**: `https://<owner>.github.io/<repo>/`
- **Branch**: `gh-pages`
- **Trigger**: On main/master branch pushes
- **Content**: `coverage.html` as `index.html`

### Codecov Integration

Coverage is also uploaded to Codecov:
- **Action**: `codecov/codecov-action@v3`
- **Flags**: `unittests`
- **Name**: `codecov-umbrella`
- **Fail on Error**: `false` (non-blocking)

## Coverage Metrics

### Tracked Metrics

1. **Statement Coverage**
   - Percentage of executable statements covered
   - Default Go coverage metric

2. **Function Coverage**
   - Percentage of functions with at least one statement covered
   - More granular than statement coverage

3. **Package Coverage**
   - Aggregated coverage per package
   - Helps identify low-coverage packages

## Coverage Best Practices

### 1. Race Detection

All tests run with `-race` flag:
- Detects data races
- Provides more accurate coverage
- Slightly slower execution

### 2. Coverage Profiles

Each test type generates its own coverage profile:
- Enables granular analysis
- Helps identify gaps in test coverage
- Supports targeted test improvements

### 3. Coverage Merging

All profiles merged into single `coverage-total.out`:
- Provides overall project coverage
- Includes all test types
- Accurate representation of test coverage

## Usage

### Local Development

Run coverage locally:
```bash
# Run all tests with coverage
go test -v -race -coverprofile=coverage.out ./...

# Generate HTML report
go tool cover -html=coverage.out -o coverage.html

# View in browser
open coverage.html
```

### CI/CD Pipeline

Coverage is automatically generated and reported on:
- Every push to main/master/develop
- Every pull request
- Manual workflow dispatch

## Troubleshooting

### Low Coverage

If coverage is below threshold:
1. Identify low-coverage packages in `coverage-func.txt`
2. Review test coverage for those packages
3. Add missing test cases
4. Re-run CI to verify improvement

### Coverage Not Merging

If coverage merge fails:
1. Check if all test jobs completed successfully
2. Verify coverage files exist in artifacts
3. Check `gocovmerge` installation
4. Review merge step logs for errors

### Coverage Badge Issues

If badge doesn't display correctly:
1. Verify SVG file generation
2. Check SVG syntax
3. Test badge in local browser
4. Verify artifact upload
